1. Architecture Overview

Everything lives in one HTML file. Internally it behaves like a tiny app with clear layers: UI (tabs + forms), logic modules (4 calculators), data (CoinGecko + localStorage), and styling (CSS variables + transitions).

UI Logic Data Styling

What files exist?

Runtime code: index.html (HTML + CSS + JS inline). The other inputs (TASK.md, notes.md) are requirements and build notes.

Offline behavior: the dashboard works without network; the only network call is the optional CoinGecko price fetch for the crypto converter. Cached prices are stored in localStorage for immediate usefulness on reload.

Design intent: this is a “single-file micro-utility”. The architecture optimizes for portability (email it, drop it in a folder, open it) while still keeping internal modules separated by convention.

2. Key Decisions

The implementation choices are deliberate: minimize moving parts, keep everything in one file, and bias for “fast to use” over “framework-perfect”.

Single-file HTML (no build step, no deps) ›
  • Why: trivially portable and offline-capable. A user can open it on iPhone, desktop, or in a restricted environment.
  • Trade-off: less modular than a multi-file project; structure is enforced by naming and sectioning inside the script.
  • Alternative: a framework app (React/Vite) would improve maintainability but violates the “single file, zero deps” constraint.
State persistence via localStorage (per-tool scopes) ›
  • Why: calculators are only useful if they remember “last used” inputs. localStorage is enough and keeps everything client-side.
  • How: a prefix key (swiss_army_calc_v1:) + scope names (crypto, poly, sizer, meeting).
  • Alternative: querystring state (shareable) was possible but would complicate the UX; this build prioritizes speed.
Live prices with debounce + manual fallback ›
  • Why: CoinGecko’s free endpoint has rate limits and can be temporarily unavailable. The tool must keep working.
  • How: fetch is limited to 1 request / 30s; if fetch fails, conversions run off manual entries (and cached values).
  • Alternative: server proxy or paid API key would improve reliability but breaks “single file, offline” and adds operational surface area.
Numbers animate on update (readability + delight) ›
  • Why: fast calculators change values frequently; animation helps the user notice what changed without hunting.
  • How: a small animateValue() helper runs requestAnimationFrame with an ease-out curve and a quick “pulse”.
  • Trade-off: slightly more code; must avoid animating non-numeric output (the build resets animation when setting raw text).

Shared utilities

parseNum() hardens inputs; fmtUSD/fmtNumber format output; animateValue() updates UI without feeling “jumpy”. These are reused across all 4 tools to keep behavior consistent.

UI composition

Tabs are just buttons + panels. The active panel gets .active and then .show to trigger an opacity/transform transition.

Persistence strategy

Each tool saves inputs on every change. On init, saved values are applied first; then initial calculations run so the UI starts “ready” instead of blank.

4. Data Flow

The “user journey” is the architecture: inputs change → state is persisted → a calc function runs → formatted output animates into place. Live crypto prices are fetched opportunistically and merged with fallback prices.

CoinGecko integration details ›
  • Endpoint used: https://api.coingecko.com/api/v3/simple/price?ids=bitcoin,ethereum,solana,matic-network&vs_currencies=usd
  • Debounce: lastFetchMs prevents calls within 30 seconds.
  • Cache: live prices and last-updated timestamp are stored in localStorage and restored on init.
  • Fallback: live mode merges valid live prices into the manual map; manual mode ignores live prices entirely.

5. Interactive Elements

This explainer itself is interactive (nav + progress + collapsibles + expandable code). The underlying build also has a few “power user” touches worth calling out.

In the dashboard (index.html)

  • Keyboard: 1–4 switches tools; Enter runs the active calculator (or toggles the meeting timer).
  • Animated outputs: numbers count up/down and “pulse” to confirm updates.
  • Crypto quick flip: swaps from/to without retyping.
  • NO Janitor mode: a one-click prefill for NO positions near 95–99¢ behavior.
